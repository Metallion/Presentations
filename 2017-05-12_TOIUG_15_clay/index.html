What problem are we solving?

Imagine a production network

These are often very complicated.

Many devices are involved.

Hubs, switches, routers, firewalls.

When one device changes, what happens?

A faulty change can result in

  • Down time
  • Security leaks

The problem can take a long time to debug, further increasing down time

How do we solve it?

The same idea as software CI

Plan => Code => Build => Test => Deploy => Repeat

But how?

Make a virtual copy of the production network

Clay

Clay is:

  • Open source
  • Written in go
  • Written by qb0C80aE
    • https://github.com/qb0C80aE/clay

Clay

Clay can:

  • Design and build the virtual networks
  • Display visual network diagrams
  • Design test cases and generate their code
  • Integrate physical devices in the virtual network

Clay

Clay uses:

  • Terraform + git
  • OpenVDC
  • OpenVNet

Integrate physical devices?

Some times a physical device needs to be tested

  • Buy another one of *only* this device
  • Simulate the rest of the network in Clay
  • Have Clay send commands to both the physical device and the virtual network

The combination of these technologies is called

Liquid Metal
